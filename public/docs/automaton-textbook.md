# オートマトン問題集のための教科書

この資料は `/automaton` の問題集に対応した「要点整理・定義・解法の型」をまとめたミニ教科書です。印刷してノートに追記できるよう、短い段落で整理しています。

---

## 学習ガイド

- **対応する問題**: 1. 基礎数理 / 2. 言語とオートマトン / 3. 計算可能性 / 4. 複雑さ / 5. 総合演習
- **進め方**
  1. 章の「定義」と「解法の型」を読む
  2. ミニ例題で手順を確認
  3. 問題集に戻って同じ手順を適用

---

## 第1章 基礎数理（対応: 1.1〜1.25）

### 集合と関数
- ド・モルガンの法則：否定の配り方を暗記。
- 単射・全射・全単射：定義域と終域を常に確認。
- 鳩の巣原理：要素数の比較で背理法。

### グラフ理論
- 握手補題：次数の総和は必ず偶数。
- オイラー閉路：全頂点が偶数次数。
- 奇数次数の頂点数は偶数個。

### 証明技法
- 背理法：否定を仮定し矛盾を導く。
- 対偶：含意を扱いやすい形に変形。
- 0で割らない：式変形の条件を確認。

### ミニ例題
- 「5頂点の次数 3,3,3,2,2 のグラフは存在するか？」
  - 次数の総和が奇数になるため存在しない。

## 第2章 言語とオートマトン（対応: 2.1〜2.x）

### 正規言語の基礎
- アルファベット Σ と言語 L ⊆ Σ*。
- DFA/NFA：状態遷移図を必ず描く。
- 閉包性：連結・和集合・反復で正規言語は保たれる。

### 典型設計パターン
- 最初の文字と長さの偶奇を分解して状態にする。
- 「開始記号を見たか」「終了記号を見たか」を分ける。
- コメント認識などは状態の直列連結で表現。

### 文脈自由言語の入口
- プッシュダウンオートマトンは「スタック」で記憶。
- {0^n 1^n} のように数を数える言語が代表例。
- 生成規則（文法）で表現できる。

### ミニ例題
- 「0で始まり長さ偶数、または1で始まり長さ奇数」
  - 開始文字(0/1)と長さ偶奇(偶/奇)を状態で保持する。

## 第3章 計算可能性（対応: 3.x）

### 主要概念
- 判定可能（decidable）：必ず停止して答える。
- 半判定可能（recognizable）：答えが「Yes」の時だけ停止。
- チャーチ＝チューリングのテーゼ：計算可能性の統一。

### 証明の型
- 対角線論法：自己参照で矛盾を作る。
- 既知の非計算可能問題からの帰着。
- 「万能な判定器があると仮定して矛盾」。

### 代表テーマ
- 停止問題（Halting Problem）。
- レンジの空性、言語等価性。
- 万能チューリング機械と符号化。

### ミニ例題
- 「停止問題が判定可能と仮定すると、自分自身を否定する機械が作れて矛盾」
  - 背理法 + 対角線論法。

## 第4章 複雑さの階層（対応: 4.1〜4.2）

### 時間計算量
- ビッグオー記法で上界を表す。
- P：多項式時間で解ける問題群。
- NP：多項式時間で「検証」できる問題群。

### NP完全性
- NP困難 + NPに属する → NP完全。
- 帰着（多項式時間変換）が鍵。
- SAT → 3SAT → CLIQUE の流れを確認。

### 空間計算量
- L, NL：対数空間で解けるか。
- PSPACE：多項式空間で解ける問題群。
- 時間と空間のトレードオフを意識。

### ミニ例題
- 「ある問題 X が NP完全と示したい」
  - 既知のNP完全問題からXへ多項式時間帰着。

## 第5章 総合演習（対応: 5.x）

### 解法チェックリスト
- 問題は「設計」か「証明」か。
- 必要な状態数 / 記憶構造は何か。
- 既知問題からの帰着が使えるか。

### よくある落とし穴
- 定義域・終域の取り違え。
- 「閉包性がある」=「自動的に構成できる」と誤解。
- 帰着で入力と出力の対応を曖昧にする。

### 仕上げ演習の提案
- 章末の問題を解いたら、同じテーマの問題を自分で作ってみましょう。
- 例: 「偶数長の回文を認識する言語」や「NP完全問題の簡易変種」を設計してみる。

## 補足: 記法と約束

- Σ* は「Σ上の全ての有限長文字列」の集合。
- L ⊆ Σ* は「言語」を表す。
- O(f(n)) は計算量の上界を表す。
- 問題番号は /automaton の問題集に合わせています。
