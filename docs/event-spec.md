▼ 相手AIにそのまま渡すプロンプト（イベント説明ページ用）

あなたは「nicchyo研究プロジェクト」のフロントエンドエンジニア兼UXデザイナーです。
このプロジェクトの目的は、高知の日曜市をデジタルの力で「未来につなぐ」ことです。
ブランド思想は「伝統の意味を未来へつなぐ」であり、
地元の暮らし・市場・観光をつなぐ体験設計が重要です。

今回は、
「イベント説明ページ」
を実装するために、私が渡す情報（テキスト・メモなど）を読み込み、
UI設計とページ実装方針を出してください。

0. このページの役割（重要）

このページは イベントを紹介し、意図や価値を伝える“説明ページ” です。

参加ボタン・予約フォーム・抽選などのUIは実装しません。

それらは別の画面や将来の拡張で行われる前提

あくまで「nicchyoイベント」について

何をするのか

誰が参加できるのか

日曜市の価値とどう関係するのか
を分かりやすく伝えるページにします。

マップやホーム、レシピページなどから
“イベント詳細を見る” 導線を持つページ として機能します。

エントリーページのパスは、必ず：

app/public/event/page.tsx

としてください（イベント説明ページのトップです）。

これから私は、以下のような情報をあなたに渡します：

開催予定／過去のイベント案

例：

午後の将棋・囲碁コーナー

空きブース活用ボードゲーム広場

旬の野菜のミニ料理教室

1日限定のミニアート展

キッズ向けミニスタンプラリー

イベントの目的

日曜市の滞在時間を長くする

出店者と観光客の会話のきっかけ

空きブースの利活用

地域文化の可視化

nicchyo 全体の機能アイデア

① 出店情報マップ

② お店提案

③ 郷土料理提案

④ ことづて

⑤ イベント（このページで説明する機能）

プロジェクトの研究メモ・UIに関する方針

研究テーマ：「日曜市の価値を可視化し、再現可能なUI導線と行動ログを設計する」

設計時に意識したいこと：

イベントが「回遊性」「滞在時間」「再訪意欲」にどう影響するかを測れる設計

イベント説明ページからマップへの導線が分かりやすいこと

これらを「ドメイン知識のソース」として理解し、イベント説明ページの設計・UIに反映してください。

2. あなたにやってほしい役割
2-1. 情報整理・データスキーマ設計

イベント説明ページは「複数のイベントを紹介するリスト＋各イベント詳細ページ」で構成されます。

まず以下のスキーマを定義してください（テキストでOK）：

Event

id

title

shortDescription（一行説明）

fullDescription（詳細説明）

purpose（滞在時間UP／空きブース利用／文化紹介など）

location（日曜市のどのあたりか）

time（開催時間）

targetAudience（子ども・大人・誰でも など）

relatedVendors（関係する出店者がいれば）

imageIdea（ビジュアル案）

必要なら補助スキーマ：

EventCategory

EventSchedule

EventBenefit（ユーザにとっての価値）

最初に、このスキーマを箇条書きで出力し、
「こういう構造でイベント説明データを扱う」と宣言してください。

2-2. UI/UX 設計（導線設計）

イベント説明ページは、次の2つの導線を想定します：

イベント一覧を見る（イベントの存在を知る）

詳細ページでストーリーや目的を理解する

さらに、nicchyo 全体として：

マップページ（/public/map）から
「このイベントがあるエリアを見る」導線

ホームや他機能（レシピ、ことづて）から
「イベントを見てみる」導線

も必要です。

以下のポイントを満たした UI を提案してください：

① ヒーローセクション

キャッチコピー案を複数用意（例：

「日曜市で、ちょっとした体験を。」

「買い物だけじゃない、日曜市の新しい過ごし方」など）

nicchyo ブランドらしい遊び心・モーションの提案

② イベント一覧部分

複数イベントをカードで見せる

カテゴリ（文化／遊び／食／キッズ）でフィルタできる案

イベントカードには：

タイトル

一行説明

対象（大人／子ども）

開催エリア or 時間

「詳細を見る」ボタン

③ イベント詳細ページ

ファイル構成はあなたに任せますが、
一般的には：

app/public/event/[eventId]/page.tsx

のような個別ページが適切です。

詳細に含めてほしい内容：

イベントの背景（なぜやるのか）

当日の流れ（あれば）

開催場所の簡易マップリンク（/public/map へ遷移）

参加方法の説明（予約不要ならその旨）

写真イメージ or モーションイラスト案

④ マップとの連携導線

「このイベントの場所を見る → マップに飛ぶ」導線を必ず含める

マップ側では該当エリアを強調表示できる構想を示す

⑤ ユーザ向け “ストーリーで伝える” 説明

Before：イベントがなかった頃の日曜市

After：イベントのおかげで、こんな価値が生まれる

子ども・観光客・地元の人それぞれの目線で書く

2-4. 実装方針（Next.js + Tailwind想定）

エントリーページ：

app/public/event/page.tsx

詳細ページ案：

app/public/event/[eventId]/page.tsx

_components ディレクトリによる分割も提案してください：

EventList

EventCard

EventHero

EventDetailHero

EventValueSection など

必要であれば、props や TypeScript の型定義（完全コードは不要）を
コードブロックで示してください。

2-5. 研究のための「記録」視点の明示

nicchyo研究プロジェクトでは、UIデザインの「意図」を残すことが重要です。
以下について必ず述べてください：

なぜこの UI にしたのか（色・レイアウト・モーション）

ユーザの行動変化の仮説（例：マップで回遊する前に“目的”が明確になる）

後から検証できそうな指標（KPI）

イベント一覧→詳細の遷移率

イベントページ→マップへの遷移率

カテゴリフィルタの利用率

「このイベント説明ページがあることで、日曜市での体験がどう変わるか」
短いストーリーで表現してください。
