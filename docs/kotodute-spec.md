▼ 相手AIにそのまま渡すプロンプト（ことづて機能・出店状況共有）

あなたは「nicchyo研究プロジェクト」のフロントエンドエンジニア兼UXデザイナーです。

【重要変更】
当初計画されていた「ことづて（メッセージ投稿）の独立ページ /public/kotodute」は **廃止** されました。
代わりに、**「ショップバナー内での出店状況（ステータス）共有機能」** として簡素化・統合されます。

この機能の目的は、SNSのようなリアルタイム交流ではなく、
「今、店が開いているか」「もう売り切れたか」といった **現地情報の最低限の共有** です。

1. 機能の役割（Shop Status Sharing）

*   **場所**: マップ上のショップバナー（店舗詳細パネル）内
*   **誰が**: 来訪者（現地にいる人）または出店者自身
*   **何を**: 現在のステータス（開店中 / 完売 / 閉店 / 準備中 など）を更新・投票
*   **なぜ**: 「行ってみたらやっていなかった」という残念な体験を減らし、安心して訪問できるようにするため。

2. UI/UX 設計方針

*   **シンプルさ**: 1タップで投票できること（例：「開いてたよ！」ボタン）。
*   **負担なし**: テキスト入力は必須としない（任意の一言メモ程度はあってもよいが、メインはステータスボタン）。
*   **信頼性**: 「最終更新時間」を表示し、情報の鮮度を伝える。
*   **位置情報（Option）**: 可能であれば、店舗の近くにいる場合のみ投票可能にするなどの制御を検討（モック段階では不要）。

3. データスキーマ案

```typescript
export type ShopStatusType = "open" | "closed" | "sold_out" | "preparing" | "unknown";

export interface ShopStatus {
  vendorId: string;
  currentStatus: ShopStatusType;
  lastUpdated: string; // ISO Date
  updatedBy: "owner" | "visitor";
  upvotes: number; // 「参考になった」「同じ状況を確認した」数
}
```

4. 実装要件

*   `ShopBanner` コンポーネント内に `StatusSection` を追加する設計を提示してください。
*   状態は Supabase 等で共有される想定ですが、まずはローカルステートまたはモックで動作する UI を作ってください。

このプロンプトでは、**独立したことづてページを作らず、マップの一部として実装する** ことを強調して回答を作成してください。
