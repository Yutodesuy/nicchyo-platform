# nicchyo の機能・取り組み一覧

## 位置づけについて（前提）

nicchyo では、すべての機能を等価には扱いません。初来訪者が抱える不安や緊張を和らげ、「体験を始められる状態」をつくることを主目的とし、それに直接寄与する機能のみを **主機能** として位置づけます。

また、来場者の情報取得行動を **「検索 → 相談（AI） → 会話（出店者）」** という段階的な流れとして捉え、デジタルと現地体験の間に生じるギャップを埋める構成を採用しています。

その他の機能は、体験を補助・拡張する **副次機能** として、必要最小限に実装します。

## 主機能（Core Features）

### ① Web版デジタルマップ（先行実装）
*   スマートフォンのブラウザから利用可能（インストール不要）。
*   日曜市全体の範囲・構造・雰囲気を直感的に把握できる設計。
*   初来訪者が「どこから歩き始めるか」を判断できる最低限の情報を提示。
*   **特徴**: 効率的な回遊や最短ルートの提示を目的としません。あくまで体験の全体像を掴み、緊張を和らげるための入口として機能させます。

### ② AI案内役「にちよさん」（相談機能）
*   ナビゲーションバー「相談」から利用可能。
*   チャットボット形式で、分からないことや不安を気軽に相談できます。
*   検索では表現しにくい、個人的・曖昧な悩みに柔軟に対応。
*   **特徴**: AIで解決できることはそこで支援し、判断しきれないことは「出店者に聞いてみる」という次の行動を後押しします。検索と現地での会話のあいだをつなぐ“翻訳者”としての役割を担います。

### ③ 検索機能（絞り込み・リスト表示）
*   ナビゲーションバー「検索」から利用可能。
*   カテゴリや条件による基本的な絞り込み検索。
*   店舗情報をリスト形式で、網羅的かつ即時に確認可能。
*   **役割**: 明確な条件・目的がある場合の「確実に知りたい」ニーズに対応します。

### ④ ショップバナー（店舗情報の最小単位）
*   マップおよび検索結果から表示される店舗ごとの簡易情報。
*   店舗名、カテゴリ、短い紹介文を中心に構成。
*   **役割**: 情報量を抑え、現地での会話や発見を妨げない設計とし、出店者と来場者の最初の接点をつくる「きっかけ」として位置づけています。

## 副機能（補助的要素）

### ⑤ レシピ情報（ショップバナー内・任意）
*   **独立したレシピページは設けません。**
*   食材を販売している店舗のショップバナー内に、任意でレシピを紐づけ可能。
*   レシピは主機能ではなく、購入後の活用イメージを補足する情報として扱います。

### ⑥ 出店状況の共有（ことづて機能の目的限定版）
*   **従来の「ことづて」投稿ページは設けません。**
*   ショップバナー内に、簡易的な出店状況表示のみを実装。
*   来訪者（店舗から一定距離内）に限り、「現在出店しているかどうか」の確認・投票が可能。
*   **目的**: 出店者に情報発信の負担をかけず、最低限の現地情報を共有し、来訪者同士で体験を妨げない範囲の情報補完を行います。

## 今回あえて実装しない機能（スコープ外）

*   **リアルタイム投稿機能**: SNS的機能は排除します。
*   **ゲーミフィケーション・イベント機能**: 体験のゲーム化は行いません。
*   **ルート最適化機能**: 効率化は目指しません。
*   **多段階・過度なレコメンド機能**: 過度な誘導は行いません。

## 機能設計の考え方（まとめ）

nicchyo では、
1.  **不安や緊張を和らげ、体験を始めるための機能を主機能として明確化**し、
2.  **楽しさや付加価値につながる要素は、副次的に最小限実装する**

という設計方針を取っています。これにより、ペルソナとのズレや機能過多による混乱を避け、初来訪者にとって最適な体験入口を提供します。
