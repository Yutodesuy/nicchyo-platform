▼ 相手AIにそのまま渡すプロンプト（レシピ情報・郷土料理）

あなたは「nicchyo研究プロジェクト」のフロントエンドエンジニア兼UXデザイナーです。

【重要変更】
当初計画されていた「独立したレシピページ /public/recipe」は **廃止** されました。
代わりに、**「ショップバナー内でのレシピ情報表示機能」** として統合されます。

この機能の目的は、レシピ検索サイトを作ることではなく、
「この野菜（食材）を買うと、こんな郷土料理が作れる」という **購入後の活用イメージ（未来の体験）** を提示し、購買のハードルを下げることです。

1. 機能の役割（Recipe Suggestion）

*   **場所**: マップ上のショップバナー（店舗詳細パネル）内
*   **トリガー**: その店舗が取り扱っている「食材」に紐づくレシピがある場合のみ表示。
*   **内容**: 料理名、写真、簡易レシピ（または外部サイト/詳細モーダルへのリンク）。

2. UI/UX 設計方針

*   **控えめに**: 店舗情報のノイズにならないよう、アコーディオンやチップ形式で表示。
*   **関連性**: 「この店で買える〇〇を使ったレシピ」という文脈を大切にする。
*   **発見**: 「これ、どうやって食べるの？」という疑問に対する、店主に聞く前の予備知識、あるいは店主に聞くきっかけとして機能させる。

3. データスキーマ案

```typescript
export interface Recipe {
  id: string;
  title: string;
  description: string; // 短い説明
  ingredients: string[]; // 必要な食材（例: ["ryukyu", "ginger"]）
  imageUrl?: string;
  cookTime?: string;
}

// Vendorデータ側
export interface Vendor {
  // ...既存フィールド
  handlingIngredients: string[]; // 取り扱い食材ID
}
```

4. 実装要件

*   `ShopBanner` コンポーネント内に、該当するレシピがある場合のみ出現する `RecipeSuggestion` セクションを設計してください。
*   レシピデータは `lib/data/recipes.ts` などの静的データとして持つ想定で設計してください。

このプロンプトでは、**独立したレシピページを作らず、マップの一部として実装する** ことを強調して回答を作成してください。
